ASSIGNMENT 1: Users, Roles & Tablespaces Management
📋 Learning Objectives
  • Create and manage Oracle database users
  • Grant system and object privileges to users
  • Create and manage database roles
  • Create and configure tablespaces
  • Add datafiles to tablespaces
  • Configure space alerts and thresholds
1
Create Three Database Users
Create three new users with restricted privileges. These users will represent different application roles with varying access levels.
-- Connect as SYSDBA CONNECT / AS SYSDBA -- Create User 1: APP_USER01 CREATE USER app_user01 IDENTIFIED BY Welcome123 DEFAULT TABLESPACE users QUOTA 100M ON users; -- Create User 2: APP_USER02 CREATE USER app_user02 IDENTIFIED BY Welcome123 DEFAULT TABLESPACE users QUOTA 100M ON users; -- Create User 3: APP_USER03 CREATE USER app_user03 IDENTIFIED BY Welcome123 DEFAULT TABLESPACE users QUOTA 100M ON users;
Expected Output:
User APP_USER01 created.
User APP_USER02 created.
User APP_USER03 created.
ℹ Note: The QUOTA clause limits the amount of space each user can consume in the USERS tablespace. This prevents accidental space exhaustion by individual users.
2
Grant CONNECT Privilege to All Users
Grant the CONNECT privilege (which includes CREATE SESSION) to enable users to connect to the database. Also grant basic CREATE privileges for app development.
-- Grant CONNECT privilege (CREATE SESSION) to all users GRANT CONNECT TO app_user01; GRANT CONNECT TO app_user02; GRANT CONNECT TO app_user03; -- Grant CREATE TABLE privilege for development GRANT CREATE TABLE TO app_user01; GRANT CREATE TABLE TO app_user02; GRANT CREATE TABLE TO app_user03; -- Grant additional resource privileges GRANT CREATE SEQUENCE TO app_user01; GRANT CREATE SEQUENCE TO app_user02; GRANT CREATE SEQUENCE TO app_user03; GRANT CREATE PROCEDURE TO app_user01; GRANT CREATE PROCEDURE TO app_user02; GRANT CREATE PROCEDURE TO app_user03;
Expected Output:
Grant succeeded. (repeated 12 times)
ℹ Note: CONNECT is a role that includes CREATE SESSION, RESOURCE, and other basic privileges. Alternatively, you can grant CREATE SESSION directly for more granular control.
3
Create a Composite Role for DML Operations
Create a custom role that bundles all the permissions needed for application users to perform INSERT, UPDATE, and DELETE operations on existing tables.
-- Create a custom role for DML operations CREATE ROLE app_dml_role; -- Grant privileges to the role GRANT CREATE SESSION TO app_dml_role; GRANT CREATE TABLE TO app_dml_role; GRANT CREATE SEQUENCE TO app_dml_role; GRANT CREATE PROCEDURE TO app_dml_role; GRANT CREATE VIEW TO app_dml_role; GRANT CREATE INDEX TO app_dml_role; GRANT CREATE TRIGGER TO app_dml_role; GRANT UNLIMITED TABLESPACE TO app_dml_role;
Expected Output:
Role created.
Grant succeeded. (repeated 8 times)
ℹ Note: UNLIMITED TABLESPACE grants the role user the ability to use unlimited space in all tablespaces. Use with caution in production; consider specific quotas instead.
4
Assign the Role to the Third User
Assign the app_dml_role to app_user03, giving this user all the bundled permissions at once. This demonstrates the power of roles for privilege management.
-- Grant the custom role to app_user03 GRANT app_dml_role TO app_user03; -- Verify the role assignment SELECT * FROM dba_role_privs WHERE grantee = 'APP_USER03';
Expected Output:
Grant succeeded.

GRANTEE GRANTED_ROLE ADMIN_OPTION
─────────────────────────────────────────────────
APP_USER03 APP_DML_ROLE NO
ℹ Note: The ADMIN_OPTION set to 'NO' means app_user03 cannot grant this role to other users. Set ADMIN_OPTION to YES if this is required.
5
Create Tablespace for User 1
Create a dedicated tablespace for app_user01. This provides storage isolation and makes it easier to manage user-specific data and performance.
-- Create tablespace for app_user01 CREATE TABLESPACE app_user01_ts DATAFILE '/oracle/oradata/app_user01_01.dbf' SIZE 100M AUTOEXTEND ON NEXT 10M MAXSIZE 500M EXTENT MANAGEMENT LOCAL SEGMENT SPACE MANAGEMENT AUTO; -- Assign as default tablespace for user1 ALTER USER app_user01 DEFAULT TABLESPACE app_user01_ts; ALTER USER app_user01 QUOTA 200M ON app_user01_ts;
Expected Output:
Tablespace created.
User altered.
User altered.
ℹ Note: AUTOEXTEND allows the tablespace to grow automatically. EXTENT MANAGEMENT LOCAL uses bitmaps for faster extent allocation.
6
Create Tablespace for User 2
Create a separate tablespace for app_user02 with the same configuration pattern as User 1.
-- Create tablespace for app_user02 CREATE TABLESPACE app_user02_ts DATAFILE '/oracle/oradata/app_user02_01.dbf' SIZE 100M AUTOEXTEND ON NEXT 10M MAXSIZE 500M EXTENT MANAGEMENT LOCAL SEGMENT SPACE MANAGEMENT AUTO; -- Assign as default tablespace for user2 ALTER USER app_user02 DEFAULT TABLESPACE app_user02_ts; ALTER USER app_user02 QUOTA 200M ON app_user02_ts;
Expected Output:
Tablespace created.
User altered.
User altered.
7
Create Tablespace for User 3
Create the final tablespace for app_user03. This user will benefit from the DML role's expanded privileges within their own tablespace.
-- Create tablespace for app_user03 CREATE TABLESPACE app_user03_ts DATAFILE '/oracle/oradata/app_user03_01.dbf' SIZE 100M AUTOEXTEND ON NEXT 10M MAXSIZE 500M EXTENT MANAGEMENT LOCAL SEGMENT SPACE MANAGEMENT AUTO; -- Assign as default tablespace for user3 ALTER USER app_user03 DEFAULT TABLESPACE app_user03_ts; ALTER USER app_user03 QUOTA 200M ON app_user03_ts;
Expected Output:
Tablespace created.
User altered.
User altered.
8
Add Secondary Datafiles to Tablespaces
Add additional datafiles to each tablespace to increase storage capacity and distribute I/O across multiple disk spindles for performance optimization.
-- Add secondary datafile to User 1 tablespace ALTER TABLESPACE app_user01_ts ADD DATAFILE '/oracle/oradata/app_user01_02.dbf' SIZE 100M AUTOEXTEND ON NEXT 10M MAXSIZE 500M; -- Add secondary datafile to User 2 tablespace ALTER TABLESPACE app_user02_ts ADD DATAFILE '/oracle/oradata/app_user02_02.dbf' SIZE 100M AUTOEXTEND ON NEXT 10M MAXSIZE 500M; -- Add secondary datafile to User 3 tablespace ALTER TABLESPACE app_user03_ts ADD DATAFILE '/oracle/oradata/app_user03_02.dbf' SIZE 100M AUTOEXTEND ON NEXT 10M MAXSIZE 500M;
Expected Output:
Tablespace altered. (repeated 3 times)
ℹ Note: Multiple datafiles allow Oracle to stripe I/O operations across different storage devices, improving query performance.
9
Set Space Usage Thresholds to 85%
Configure space alerts on all tablespaces to trigger warnings when they reach 85% capacity. This provides proactive monitoring to prevent running out of space.
-- Set warning threshold at 85% for User 1 tablespace ALTER TABLESPACE app_user01_ts STORAGE (MAXEXTENTS UNLIMITED); -- Create alert for app_user01_ts at 85% BEGIN DBMS_SPACE_ADMIN.TABLESPACE_THRESHOLD_ALERT( ts_name => 'APP_USER01_TS', threshold_type => DBMS_SPACE_ADMIN.THRESHOLD_PERCENT, threshold_value => 85, alert_type => DBMS_SPACE_ADMIN.ALERT_ON_PERCENT ); END; / -- Set warning threshold at 85% for User 2 tablespace BEGIN DBMS_SPACE_ADMIN.TABLESPACE_THRESHOLD_ALERT( ts_name => 'APP_USER02_TS', threshold_type => DBMS_SPACE_ADMIN.THRESHOLD_PERCENT, threshold_value => 85, alert_type => DBMS_SPACE_ADMIN.ALERT_ON_PERCENT ); END; / -- Set warning threshold at 85% for User 3 tablespace BEGIN DBMS_SPACE_ADMIN.TABLESPACE_THRESHOLD_ALERT( ts_name => 'APP_USER03_TS', threshold_type => DBMS_SPACE_ADMIN.THRESHOLD_PERCENT, threshold_value => 85, alert_type => DBMS_SPACE_ADMIN.ALERT_ON_PERCENT ); END; /
Expected Output:
Tablespace altered.
PL/SQL procedure successfully completed. (repeated 3 times)
⚠ Warning: Alerts are triggered when threshold is reached. Monitor the alert log and respond promptly to prevent tablespace full errors.
